## 1. 画面一覧と必要な権限

左メニューに表示される画面は、**付与されているロール（役割）によって異なります**。  
権限がない画面はメニューに表示されません。

| # | 画面 | メニュー名 | 主な利用者 | 表示に必要な権限（いずれか） |
|---|------|-----------|-----------|------------------------------|
| 1 | ダッシュボード | ダッシュボード | 全員 | ダッシュボード表示権限 |
| 2 | 一覧 | 経費申請一覧 | 申請者・承認者・経理 | 自分／部門／全社の参照権限のいずれか |
| 3 | 登録 | 経費申請登録 | 申請者 | 経費申請の作成権限 |
| 4 | 詳細 | （サブ画面） | 全員 | 一覧と同様の参照権限 |
| 5 | 承認・精算 | 承認・精算管理 | 承認者・経理 | 承認、または精算処理のいずれか |

**標準ロールと画面の対応（目安）**

| ロール名 | 主な業務 | 見えるメニュー（目安） |
|----------|----------|------------------------|
| 経費申請者 | 自分の経費を申請 | ダッシュボード、一覧、登録 |
| 経費承認者 | 部門の申請を承認 | ダッシュボード、一覧、承認・精算（承認タブ） |
| 経費精算担当 | 精算処理・会計連携 | ダッシュボード、一覧、承認・精算（精算管理タブ） |
| 経費精算管理者 | 上記すべて＋設定 | すべてのメニュー・操作 |
| 一般メンバー（兼務） | 申請のみ | 申請者と同様 |
| 経理担当者（兼務） | 精算・freee 連携 | 精算担当と同様 |

**左メニューにない共通部品**

- 左サイドバー下部：ログインユーザー情報、freee 事業所選択、プロファイル再取得
- 画面下部付近：操作結果のトースト通知（数秒で消える）
- 各種確認ダイアログ：提出・承認・削除・精算・freee 連携など

## 2. 起動・共通操作

### 2.1 アプリの開き方

1. Agentify プラットフォームにログインする
2. **経費精算** アプリを開く
3. ユーザー情報と権限が読み込まれる（通常は数秒）
4. 権限に応じた左メニューが表示される

**メニューが1つも表示されない場合**  
→ 経費精算のロールが付与されていない可能性があります。上司またはシステム管理者に連絡してください（[権限・ロール説明書](./role-permission-guide.md) 参照）。

### 2.2 共通レイアウト

```
┌──────────────┬─────────────────────────────┐
│ ロゴ         │                             │
│ 経費精算     │      メインコンテンツ        │
├──────────────┤                             │
│ メニュー     │                             │
│ ・ダッシュ…  │                             │
│ ・経費申請…  │                             │
│  …           │                             │
├──────────────┤                             │
│ freee・ユーザー│                           │
└──────────────┴─────────────────────────────┘
```

### 2.3 左サイドバー下部（freee・ユーザー情報）

| 要素 | 説明 |
|------|------|
| 緑／黄の丸 | freee 連携状態（緑＝連携済み、黄＝未連携など） |
| 事業所プルダウン | freee に登録する事業所を選択 |
| 回転矢印（freee） | freee から状態を同期（権限がある場合のみ） |
| 建物アイコン | 事業所一覧を再取得 |
| freee認証 | トークン未取得・期限切れ時に表示。クリックで認証画面へ |
| ユーザー名・メール | ログイン中のユーザー |
| 回転矢印（ユーザー） | プロファイル・部門・承認者情報を再取得 |

**再取得のタイミング**

- 氏名が「…」のまま変わらない
- 部門や承認者が古いまま
- freee の事業所が選べない

### 2.4 確認ダイアログ（共通）

| 操作 | ダイアログの内容 |
|------|------------------|
| 申請提出 | 「経費申請を提出しますか？」→ **提出する** |
| 承認 | 「この申請を承認しますか？」→ **承認する** |
| 差戻し | 差戻し理由を入力 → **差戻す** |
| 削除 | 「この申請を削除しますか？」→ **削除する** |
| 取下げ | 確認なしで実行（申請中のみ） |
| 精算処理 | 「精算済みにしますか？」→ **精算済みにする** |
| 精算済み | 「精算済みにしますか？」→ **精算済みにする** |
| Excel 生成 | 「Excel を生成しますか？」→ **生成する** |
| freee 連携 | 「freee に取引として登録します。よろしいですか？」→ **連携する** |

### 2.5 トースト通知

保存・提出・承認・エラーなどが画面上部または下部に一時表示されます。  
例：「経費申請を提出しました」「勘定科目・税区分を設定してください」

## 3. ダッシュボード

**メニュー**：ダッシュボード  
**画面タイトル**：ダッシュボード  
**目的**：今月の申請状況、承認待ち、精算状況をひと目で確認する

### 3.1 前提（権限）

| 条件 | 内容 |
|------|------|
| 必要な権限 | ダッシュボード表示権限（経費精算管理者は常に可） |
| データの見え方 | 画面上の KPI・一覧は、**ログインユーザーのメールアドレスに紐づく申請**を集計（現行 API：`keihi_list_expenses`）。部門・全社参照権限はメニュー表示に使われますが、一覧取得の範囲拡張は今後対応予定です |

### 3.2 画面構成

```
┌─ ヘッダー ─────────────────────────────────┐
│ ダッシュボード                              │
├─ KPIカード（5つ） ─────────────────────────┤
│ 今月の申請金額 / 承認待ち件数 / 差戻し件数   │
│ 精算待ち金額 / 精算済み金額                  │
├─ 自分の申請一覧（最大5件） ─┬─ 承認待ち一覧 ─┤
└────────────────────────────┴────────────────┘
```

### 3.3 KPI の意味

| KPI | 意味 |
|-----|------|
| 今月の申請金額 | 今月「申請日」が当月の申請金額の合計 |
| 承認待ち件数 | ステータスが「申請中」の件数 |
| 差戻し件数 | ステータスが「差戻し」の件数 |
| 精算待ち金額 | ステータスが「精算待ち」の金額合計 |
| 精算済み金額 | ステータスが「精算済み」の金額合計 |

### 3.4 操作手順

**自分の申請を確認する**

1. 左下「自分の申請一覧」テーブルを見る
2. 行をクリック → **経費申請詳細** へ遷移
3. 「すべて見る」→ **経費申請一覧** へ遷移

**承認待ちを確認する（承認者向け）**

1. 右側「承認待ち一覧」を見る
2. 行をクリック → 詳細画面で承認・差戻し
3. 「すべて見る」→ **承認・精算管理** へ遷移

## 4. 経費申請一覧

**メニュー**：経費申請一覧  
**画面タイトル**：経費申請一覧  
**目的**：登録済みの経費申請を検索・確認し、一括操作する

### 4.1 前提（権限）

| 操作 | 必要な権限 |
|------|-----------|
| 画面を開く | 自分／部門／全社の参照権限のいずれか |
| 新規経費申請 | 経費申請の作成権限 |
| Excel を生成 | Excel 出力権限 |
| freee に連携 | freee 取引連携権限 |
| 行の操作（編集・承認など） | ステータスとロールに応じた個別権限 |

**参照範囲の目安**

| 権限の種類 | 一覧に表示される申請（設計上） | 現行実装 |
|-----------|-------------------------------|----------|
| 自分のみ | 自分が申請したものだけ | ログインユーザーのメールに紐づく申請 |
| 所属部門 | 同じ部門の申請 | 同上（部門フィルターは画面上で利用可） |
| 全社 | テナント内のすべての申請 | 同上（API 拡張予定） |

### 4.2 画面構成

```
┌─ ヘッダー ─────────────────────────────────────────┐
│ 経費申請一覧    [Excel] [freee連携] [新規経費申請]  │
├─ 検索フィルター（1段目） ───────────────────────────┤
│ ステータス / 申請者 / 経費科目 / キーワード          │
├─ 検索フィルター（2段目） ───────────────────────────┤
│ 申請日 From-To / 利用日 From-To / 連携状態（freee）  │
├─ 一覧テーブル ──────────────────────────────────────┤
│ □ 申請番号 … ステータス freee連携状態 操作           │
└────────────────────────────────────────────────────┘
```

### 4.3 検索の操作

**1段目**

1. **ステータス** … 下書き、申請中、差戻し、精算待ち、精算済み などで絞り込み
2. **申請者** … 申請者名で絞り込み
3. **経費科目** … 交通費、会議費 などで絞り込み
4. **キーワード** … 申請番号、タイトル、申請者名などを部分一致検索

**2段目**

5. **申請日（From / To）** … 申請日の期間で絞り込み
6. **利用日（From / To）** … 経費発生日の期間で絞り込み
7. **連携状態（freee）** … 次のいずれかで絞り込み
   - **連携待ち** … 精算待ち以降かつ勘定科目・税区分が設定済みで、まだ freee 未連携
   - **連携済み** … freee 取引として登録済み
   - **未連携** … 未承認、または勘定科目・税区分未設定、または連携対象外

「未連携」を選ぶと、未承認・設定不足の申請も含まれる旨のトーストが表示されます。

フィルターはすべて組み合わせ可能です。

### 4.4 一覧の列

| 列 | 説明 |
|----|------|
| □ | 一括操作用チェック（このページのみ全選択可） |
| 申請番号 | 自動採番された ID |
| 申請タイトル | 申請の件名 |
| 申請者 | 申請した社員名 |
| 部門 | 費用負担部門 |
| 申請日 | 申請を登録した日 |
| 利用日 | 経費が発生した日 |
| 経費科目 | 交通費、会議費 など |
| 金額 | 税込金額（表示形式：¥1,234） |
| ステータス | 下書き〜精算済み |
| freee 連携 | **連携済み**／**連携待ち**／**未連携**（ステータスと勘定科目・税区分から自動判定） |
| 操作 | 状況に応じたボタン（編集、承認、詳細 など） |

行をクリックすると **経費申請詳細** が開きます。

### 4.5 一括操作（上部ボタン）

**前提**：一覧で1件以上チェックを入れる。権限がないボタンは表示されません。

#### Excel を生成

1. 対象行にチェックを入れる
2. **Excel を生成** をクリック
3. 確認ダイアログで **生成する**
4. 完了メッセージを確認（Agent 連携時はファイル生成を依頼）

#### freee に連携（一括）

**必要な権限**：freee 取引連携権限（経理担当・精算担当・管理者など）

1. **精算待ち**、**精算済み** の申請が対象（下書き・申請中・差戻しは不可）
2. **すでに連携済み**の申請はスキップされる
3. **勘定科目・税区分**が未設定の申請は連携されず、スキップ件数に含まれる
4. チェックを入れて **freee に連携** → 確認 → **連携する**
5. 複数件の場合、成功・失敗・対象外の件数がトーストで報告される

#### 新規経費申請

1. **新規経費申請** をクリック
2. **経費申請登録** 画面へ遷移

### 4.6 行ごとの操作（操作列）

申請者本人か、承認者か、ステータスによってボタンが変わります。

| 状況 | 表示される操作（例） |
|------|---------------------|
| 自分の下書き | 編集、削除 |
| 自分の申請中 | 取下げ、詳細 |
| 自分の差戻し | 編集、再申請 |
| 他人の申請中（承認者） | 承認、差戻し、詳細 |
| その他 | 詳細、Excel、freee（権限・状態による） |

**取下げ** … 申請中の自分の申請を下書きに戻す（「申請取消」ではなく **取下げ** と表示）

## 5. 経費申請登録（領収書から作成）

**メニュー**：経費申請登録  
**画面タイトル**：領収書から経費申請を作成  
**目的**：領収書・請求書をアップロードし、経費申請を作成して提出する

### 5.1 前提（権限）

| 操作 | 必要な権限 |
|------|-----------|
| 画面を開く | 経費申請の作成権限 |
| 下書き保存 | 作成権限 |
| 申請提出 | 作成権限 ＋ 申請提出権限 |
| 提出前の必須条件 | **勘定科目・税区分**（freee 連携用）が設定済みであること。テナント既定値・マスタマッピングで自動解決される場合もあります |

承認者はマスタ登録に基づき自動表示され、申請者は通常変更できません。

**AI 読取時の freee 連携**

- テナントの freee 認証が **連携済み**（`authStatus = connected`）かつ freee API が利用可能な場合、AI 処理前に freee の **勘定科目一覧**・**税区分一覧** を取得してプロンプトに注入します
- AI が経費科目に加え、**勘定科目 ID**・**税区分コード** を候補として返すことがあります（未設定の場合のみ自動保存）
- freee 未連携・未認証の場合は、経費科目など基本項目のみ AI が入力し、勘定科目・税区分はスキップされます

### 5.2 画面の流れ（3ステップ）

```
ステップ1 アップロード  →  ステップ2 内容確認  →  ステップ3 保存・申請
   領収書・請求書            フォーム編集              下書き / 提出
```

### 5.3 ステップ1：領収書のアップロード

1. 点線枠内をクリックするか、ファイルをドラッグ＆ドロップ
2. 対応形式：画像（JPEG 等）、PDF
3. **1枚の領収書につき1件の経費申請**が作成される
4. 複数枚アップロードすると、申請番号が DB 採番（`keihi_allocate_application_numbers`）で自動付与される
5. AI が読み取り、金額・日付・支払先・**経費科目** などをフォームに反映（読み取り中は待機）
6. freee 連携済みの場合は **勘定科目・税区分** も AI が候補入力（後述）

### 5.4 ステップ2：内容の確認・編集

各領収書が **経費申請ブロック** として表示されます。

| 項目 | 必須 | 説明 |
|------|------|------|
| 申請タイトル | ○（提出時） | 申請の件名 |
| 部門 | ○ | 費用負担部門（所属部門から選択） |
| 承認者 | 自動 | 部門に応じて自動設定 |
| 申請日 | ○ | 申請する日（既定：今日） |
| 利用日 | ○ | 経費が発生した日 |
| 経費科目 | ○ | 交通費、会議費 など（AI が固定候補から選択） |
| 勘定科目 | — | 会計上の科目（freee 連携時は AI または詳細画面で設定） |
| 内容・支払先・支払方法 | — | 領収書の内容 |
| 金額 | ○ | 税込金額 |
| 税区分・税率 | — | 税込／税抜、10% など |
| 適格請求書 | — | あり／なし／不明 |

**ブロックごとの保存**

1. 内容を確認・修正する
2. 各ブロックの **このブロックを保存** をクリック
3. 「保存済み」バッジが付くまで繰り返す  
   ※ 未保存のブロックがあると下書き・提出できません

### 5.5 ステップ3：下書き保存・申請提出

画面下部の固定バーから操作します。

#### 下書き保存

1. すべてのブロックを「保存済み」にする
2. **下書き保存** をクリック
3. 「○件の下書きを保存しました」と表示
4. 後から **経費申請一覧** で編集・提出可能

#### 申請提出

1. すべてのブロックを保存済みにする
2. 申請タイトルが未入力のブロックがないことを確認
3. **勘定科目・税区分**（freee 用）が設定済みであること（未設定時はエラー）
4. **申請提出** をクリック
5. 確認ダイアログで **提出する**
6. ステータスが **申請中** になり、承認者の承認待ちとなる

## 6. 経費申請詳細

**メニュー**：（一覧やダッシュボードから遷移）  
**画面タイトル**：申請番号＋申請タイトル  
**目的**：1件の申請内容を確認し、承認・精算・freee 連携を行う

### 6.1 前提（権限）

| 操作 | 必要な権限・条件 |
|------|-----------------|
| 画面を開く | 参照権限の範囲内の申請のみ |
| 編集・提出・取下げ・削除 | 自分の申請 ＋ 該当操作権限 |
| 承認・差戻し | 承認権限 ＋ 他人の「申請中」 |
| 精算済み更新 | 精算処理権限 |
| freee に連携 | freee 取引連携権限（一覧・詳細の個別／一括） |

### 6.2 画面構成

```
┌─ 戻るリンク ───────────────────────────────┐
├─ ヘッダー（番号・タイトル・金額・ステータス）─┤
│  [freee に連携]  [承認/提出/精算 など]        │
├─ タブ ───────────────────────────────────────┤
│  基本情報 | 経費情報 | 添付ファイル | 履歴    │
├─ タブ内容 ───────────────────────────────────┤
│  （freee 必須設定エリア：基本・経費タブ）     │
└──────────────────────────────────────────────┘
```

### 6.3 freee 取引連携の必須設定

**基本情報** または **経費情報** タブの上部に表示されます。

| 項目 | 説明 |
|------|------|
| 勘定科目 | freee の勘定科目（プルダウン） |
| 税区分 | freee の税区分（プルダウン） |

**設定手順**

1. 詳細画面を開くと、可能な場合は自動でマスタを取得
2. 取得できない場合は **マスタ取得** をクリック
3. **勘定科目** と **税区分** を選択（選択すると自動保存）
4. **自動設定** … 経費科目から勘定科目を推測して入力

**いつ必要か**

- 申請を **提出** する前
- **freee に連携** する前

未設定のまま提出しようとすると、エラーメッセージが表示されます。

### 6.4 ステータス別の操作（ヘッダー右側）

#### 下書き（自分の申請）

| ボタン | 操作 |
|--------|------|
| 編集 | 登録画面と同様の編集へ |
| 申請する | 提出確認 → 申請中へ |
| 削除 | 削除確認 → 一覧から消える |

#### 申請中

| 誰 | ボタン |
|----|--------|
| 申請者本人 | **取下げ**（下書きに戻す） |
| 承認者 | **承認**、**差戻し**（理由入力） |

#### 差戻し（自分の申請）

| ボタン | 操作 |
|--------|------|
| 編集 | 内容を修正 |
| 再申請 | 修正後に再度申請中へ |

#### 精算待ち

承認後、申請は精算処理の対象として **精算待ち** に進みます。

| ボタン | 操作者 |
|--------|--------|
| 精算済みにする | 精算処理権限を持つ担当者（精算済みへ） |

※ 精算担当が支払・経理処理の完了後に **精算済みにする** を押して記録を残します。

### 6.5 freee に連携

**条件（すべて満たすこと）**

- ステータスが **精算待ち**、**精算済み** のいずれか
- まだ freee に連携していない
- 勘定科目・税区分が設定済み
- freee 連携権限がある

**手順**

1. **freee に連携** をクリック
2. 確認ダイアログで **連携する**
3. 成功すると「連携済み」表示になり、**同じ申請は再連携できません**

### 6.6 タブの内容

| 基本情報 | 申請者、部門、承認者、申請日、利用日、備考 |
| 経費情報 | 科目、金額、税情報、支払情報 など |
| 添付ファイル | 領収書・補足資料の一覧 |
| 承認・精算履歴 | 提出、承認、差戻し、精算などの履歴 |

## 7. 承認・精算管理

**メニュー**：承認・精算管理  
**画面タイトル**：承認・精算管理  
**目的**：承認待ち・精算待ち・精算済みの申請をまとめて処理する

### 7.1 前提（権限）

| タブ | 必要な権限 |
|------|-----------|
| 承認待ちタブを使う | 承認権限（申請者本人の申請はこの画面では承認しない） |
| 精算管理タブを使う | 精算処理権限 |

メニュー自体は、上記いずれかの権限があれば表示されます。

### 7.2 画面構成

```
┌─ ヘッダー ─────────────────────────────────┐
│ 承認・精算管理                              │
├─ タブ ─────────────────────────────────────┤
│  [承認待ち (件数)]  [精算管理 (件数)]       │
├─ タブ内容 ─────────────────────────────────┤
│  承認待ち: 一覧 + 承認/差戻し               │
│  精算管理: 「精算待ち」+「精算済み（記録）」│
└────────────────────────────────────────────┘
```

### 7.3 承認待ちタブ

**対象**：ステータスが「申請中」で、**自分以外**が申請したもの

**承認手順**

1. **承認・精算管理** → **承認待ち** タブ
2. 対象行の **承認** をクリック
3. 確認ダイアログで **承認する**
4. ステータスが **精算待ち** になり、精算担当の処理対象になります

**差戻し手順**

1. **差戻し** をクリック
2. 差戻し理由を入力（例：「領収書の宛名を確認してください」）
3. **差戻す**
4. 申請者に **差戻し** ステータスで戻る

**自分の申請中がある場合**

画面上部に注意メッセージが表示されます。  
自分の申請の **取下げ** は **経費申請一覧** から行ってください。

### 7.4 精算管理タブ

精算管理タブは **2 つのブロック** に分かれています。

#### 精算待ち

**対象**：ステータスが「精算待ち」の申請

| 操作 | 説明 |
|------|------|
| 精算済みにする | 支払・経理処理が完了したらクリック（**精算済み** へ） |
| 詳細 | 申請内容の確認 |

※ 承認後、対象申請は精算待ちとして表示されます。精算担当は内容を確認し、支払・経理処理が完了したら **精算済みにする** を実行します。

#### 精算済み（記録）

**対象**：ステータスが「精算済み」の申請（参照・履歴確認用）

| 操作 | 説明 |
|------|------|
| 詳細 | 申請内容・freee 連携状態の確認 |

このブロックでは **精算済みにする** 操作は行いません（すでに完了済みの記録表示です）。

### 7.5 一覧からの Excel・freee

承認・精算画面の各行にも、権限があれば **Excel**・**freee** ボタンが表示されます（一覧画面と同様のルール）。

## 8. 申請ステータスの意味

| ステータス | 意味 | バッジ色（目安） | 次にできること |
|-----------|------|-----------------|---------------|
| 下書き | 作成中。まだ承認者に回っていない | 灰 | 編集、削除、提出 |
| 申請中 | 承認者の確認待ち | 黄 | 取下げ（本人）、承認・差戻し（承認者） |
| 差戻し | 承認者が修正依頼 | 赤 | 編集、再申請 |
| 精算待ち | 承認後、精算担当の処理待ち | 紫 | 精算済みに更新、freee 連携 |
| 精算済み | 精算完了（記録） | 緑 | 参照、freee 連携（未連携の場合） |

**freee 連携ができるステータス**：精算待ち、精算済み（いずれも **1回のみ** 連携可）

**freee 連携状態（一覧の表示ラベル）**

| 表示 | 意味 |
|------|------|
| 連携済み | freee 取引として登録済み |
| 連携待ち | 精算待ち以降かつ勘定科目・税区分設定済みで、未連携 |
| 未連携 | 未承認、または設定不足、または連携対象外 |

## 9. 業務フロー早見表

### 9.1 申請者（一般社員）の流れ

```
領収書を用意
    ↓
経費申請登録（アップロード → 内容確認 → ブロック保存）
    ↓
勘定科目・税区分を設定（詳細画面でも可）
    ↓
申請提出 → 申請中
    ↓
承認される → 精算待ち
    ↓
（精算担当が精算済みにする → 精算済み）
    ↓
（必要に応じて）一覧から freee 一括連携、または詳細から個別連携
```

**差戻しされた場合**

```
差戻し通知を確認
    ↓
一覧または詳細で編集
    ↓
再申請 → 申請中
```

### 9.2 承認者（部門責任者など）の流れ

```
ダッシュボード or 承認・精算管理で承認待ちを確認
    ↓
内容・領収書を確認
    ↓
問題なければ承認 / 修正が必要なら差戻し（理由記載）
```

### 9.3 精算・経理担当の流れ

```
精算待ちの申請を確認
    ↓
（支払・会計処理を実施）
    ↓
承認・精算管理の「精算管理」タブ、または詳細画面で「精算済みにする」
    ↓
精算済み（記録として一覧に残る）
    ↓
一覧で複数選択 → freee 一括連携（権限・設定が揃っている場合）
    または詳細画面から個別に freee 連携
```

### 9.4 日次の確認ポイント

| 役割 | 確認すること |
|------|-------------|
| 申請者 | 差戻しがないか（ダッシュボードの差戻し件数） |
| 承認者 | 承認待ち件数（ダッシュボード・承認・精算管理） |
| 経理 | 精算待ちの件数・金額（ダッシュボード・承認・精算管理） |

## 10. よくある質問

### Q1. メニューが少ない／画面が開けない

付与されているロールによってメニューが変わります。  
必要な業務ができない場合は、上司またはシステム管理者にロール付与を依頼してください。  
→ [権限・ロール説明書](./role-permission-guide.md)

### Q2. 申請提出できない（勘定科目のエラー）

**勘定科目** と **税区分** の両方を設定してください。  
経費申請詳細の「freee 取引連携の必須設定」で選択できます。

### Q3. freee に連携できない

次を確認してください。

- ステータスが精算待ち・精算済みのいずれかか
- 勘定科目・税区分が設定済みか（一覧の連携状態が「連携待ち」になっているか）
- すでに連携済みでないか
- 左下の freee 認証・事業所選択が正しいか
- freee 連携の権限があるか（通常は経理・精算担当・管理者）

### Q4. 一覧の「連携状態」と「ステータス」の違い

- **ステータス** … 申請・承認・精算の業務進捗（下書き〜精算済み）
- **連携状態（freee）** … freee への会計連携の readiness（連携済み／連携待ち／未連携）。未承認や勘定科目未設定の申請は「未連携」になります

### Q5. 自分の申請を承認・精算管理で承認できない

**自分の申請は自分で承認できません**（申請中の場合は一覧から **取下げ**）。

### Q6. 承認者が自動で変わる／選べない

承認者は **部門マスタ** に基づき自動設定されます。変更が必要な場合はシステム管理者へ（マスタ情報管理の部門設定）。

### Q7. Excel が生成されない

Excel 出力権限が付与されているか確認してください。

### Q8. 削除した申請を戻したい

削除した申請は **元に戻せません**。新規に作り直してください（下書きのみ削除可能）。
