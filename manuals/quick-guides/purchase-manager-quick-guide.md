# 購買担当・承認者向け担当別手順

対象者: 購買担当者、購買依頼承認者、発注案件承認者

最終更新日: 2026-06-17

## 1. 発注管理を開く

1. Agentify にログインします。
2. 「マイアプリ」を開きます。
3. 「発注管理」をクリックします。
4. 左メニューから「購買依頼」「発注案件管理」「発注管理」を確認します。

![マイアプリから発注管理を開く](../assets/common/agentify-my-apps-purchase.png)

## 2. 表示される主な画面

| 画面 | 用途 |
| --- | --- |
| 購買依頼 | 依頼内容の確認、承認、差戻し、発注案件への変換 |
| 発注案件管理 | 発注案件の作成、確認、承認、発注登録 |
| 発注管理 | 発注済みデータ、発注書、納品予定の確認 |
| 仕入先管理 | 仕入先情報の確認 |

検収登録や支払済み更新が表示されない場合があります。これは検収担当者または経理担当者の業務です。

![発注管理責任者の発注案件詳細](../assets/roles/purchase-manager-project-detail.png)

## 3. 購買依頼を承認・差戻しする

1. 購買依頼一覧を開きます。
2. ステータスが「申請中」の依頼を確認します。
3. 内容、金額、希望納期、部門を確認します。
4. 問題なければ「承認」をクリックします。
5. 修正が必要な場合は「差戻し」をクリックします。

承認後、必要に応じて「発注案件に変換」を行います。

![購買依頼承認確認](../assets/purchase/request-approve-confirm.png)

## 4. 発注案件を作成・申請する

1. 発注案件管理を開きます。
2. 「新規発注案件」をクリックします。
3. 発注案件名、仕入先、金額、納期、支払予定日、明細を入力します。
4. 必要に応じて「資料アップロード・AI抽出」を使います。
5. 内容を確認し、「申請する」をクリックします。

AI抽出を使った場合も、仕入先、数量、単価、税額、合計金額は必ず確認してください。

![新規発注案件](../assets/purchase/project-new.png)

## 5. 発注案件を承認・発注登録する

1. 発注案件詳細を開きます。
2. ステータスが「申請中」の場合、内容を確認して「承認する」または「差戻し」を行います。
3. ステータスが「承認済」になったら「発注登録」をクリックします。
4. 確認ポップアップで内容を確認し、発注レコードを作成します。

主なステータス:

| ステータス | 意味 |
| --- | --- |
| 申請中 | 承認待ちです。 |
| 承認済 | 発注登録できる状態です。 |
| 発注済 | 発注レコードが作成されています。 |
| 検収済 | 検収が完了しています。 |
| 仕入計上済 | 仕入と支払予定が作成されています。 |
| 完了 | 支払まで完了しています。 |

![発注案件詳細](../assets/purchase/project-detail.png)

## 6. 発注書を印刷・PDF保存する

1. 発注一覧から発注詳細を開きます。
2. 「発注書を印刷」をクリックします。
3. プレビューで仕入先、納品予定日、明細、金額を確認します。
4. 「印刷・PDF保存」をクリックします。

発注書は発注済みデータをもとに出力します。下書きや申請中の発注案件を発注書として扱わないでください。

![発注書印刷ボタン](../assets/purchase/purchase-order-print-button.png)
