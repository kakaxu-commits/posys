# 権限・ロール説明書

対象読者: システム管理者、部門責任者、権限設定を確認する担当者

最終更新日: 2026-06-16

## 1. 基本方針

本システムでは、ユーザーにロールを割り当てることで、表示できるメニューと実行できる操作を制御します。

- 複数ロールを持つ場合、権限は加算されます。
- 弱いロールを追加しても、既に持っている強い権限は取り消されません。
- ロールには有効期間を設定できます。有効期間外のロールは利用できません。
- 部門範囲は、ユーザーの主所属部門、兼務部門、ロール割当部門です。
- 親部門に所属していても、子部門が自動で参照範囲に含まれるわけではありません。
- システム管理者ロールは、受注管理・発注管理・マスタ情報管理で管理者として扱われます。

![ロール権限編集画面](./assets/master/master-role-permissions.png)

## 2. 標準ロール

| ロール | 主な利用者 | 受注管理 | 発注管理 | マスタ情報管理 |
| --- | --- | --- | --- | --- |
| システム管理者 | 管理者 | 全操作 | 全操作 | 全操作 |
| 受注管理責任者 | 営業責任者 | 担当部門の参照・作成・更新 | 利用なし | 利用なし |
| 発注管理責任者 | 購買責任者 | 利用なし | 購買依頼、発注案件、発注、支払予定参照 | 利用なし |
| 経理担当者 | 経理担当 | 全社受注参照、請求、売掛、入金 | 支払予定参照、支払予定更新 | 利用なし |
| 一般メンバー | 一般利用者 | 担当部門参照、受注作成 | 担当部門参照、購買依頼作成・更新 | 利用なし |

### 2.1 初期推奨権限配置

初期導入時は、まずマスタ情報管理の「標準ロール設定を適用」で以下の権限構成を作成することを推奨します。
この操作を実行すると、標準ロールの権限は推奨テンプレートに同期され、テンプレート外の追加権限は削除されます。

| ロール | 推奨権限 |
| --- | --- |
| システム管理者 | マスタ管理の全操作、受注管理の全操作、発注管理の全操作 |
| 受注管理責任者 | 受注ダッシュボード表示、自部門の受注参照、受注作成、受注更新 |
| 発注管理責任者 | 発注ダッシュボード表示、自部門の発注参照、購買依頼作成、購買依頼更新・申請、購買依頼承認・差戻し、発注案件作成、発注案件更新、発注案件承認・差戻し、仕入先参照、支払予定参照 |
| 経理担当者 | 受注ダッシュボード表示、全社受注参照、請求参照、請求処理、売掛台帳参照、回収参照、入金処理、支払予定参照、支払予定処理 |
| 一般メンバー | 自部門の受注参照、受注作成、自部門の発注参照、購買依頼作成、購買依頼更新・申請 |

補足:

- システム管理者ロールを持つユーザーは、ロール自体でも管理者として扱われます。
- 受注管理の全操作権限と発注管理の全操作権限は強い権限です。一般利用者向けロールには付与しないでください。
- 受注承認は予約権限です。現行の受注管理では承認フローを使用していないため、通常は付与しません。
- 旧形式の発注互換権限は、新規ロールでは通常使用しません。購買依頼用と発注案件用の個別権限を優先してください。
- 標準ロールを自社用に大きく変更したい場合は、標準ロールを直接変更するよりも、専用ロールを追加してユーザーへ割り当てる運用を推奨します。

### 2.2 職務分離する場合の推奨ロール例

標準ロールより細かく業務を分けたい場合は、以下のようにロールを追加します。

| 追加ロール例 | 主な利用者 | 推奨権限 | 注意点 |
| --- | --- | --- | --- |
| 受注作成担当 | 営業担当 | 自部門の受注参照、受注作成 | 下書き保存や登録後編集も必要な場合は、受注更新も追加 |
| 受注更新担当 | 営業責任者 | 受注ダッシュボード表示、自部門の受注参照、受注作成、受注更新 | 部門内の受注確定・編集まで担当 |
| 請求担当 | 請求担当者 | 全社受注参照、請求参照、請求処理、売掛台帳参照 | 入金処理も担当する場合は回収権限を追加 |
| 入金担当 | 入金担当者 | 全社受注参照、売掛台帳参照、回収参照、入金処理 | 請求取消をさせない場合は、請求処理を付与しない |
| 購買依頼者 | 一般利用者 | 自部門の発注参照、購買依頼作成、購買依頼更新・申請 | 作成だけでよい場合は、購買依頼更新・申請を外す |
| 購買依頼承認者 | 部門責任者 | 自部門の発注参照、購買依頼承認・差戻し | 承認後に案件化も行う場合は、発注案件作成を追加 |
| 発注案件担当 | 購買担当者 | 発注ダッシュボード表示、自部門の発注参照、発注案件作成、発注案件更新、仕入先参照 | 発注登録まで担当する想定 |
| 発注案件承認者 | 購買責任者 | 自部門の発注参照、発注案件承認・差戻し | 案件の編集も行う場合は、発注案件更新を追加 |
| 検収担当 | 現場担当者 | 自部門の発注参照、検収処理 | 検収後の仕入計上はできません |
| 支払担当 | 経理担当者 | 支払予定参照、支払予定処理 | 発注案件や検収の確認まで必要な場合は、別途参照ロールを追加 |

最小権限で始める場合は、作成・承認・経理処理を別ロールに分け、必要になった時点で参照範囲や更新権限を追加してください。

### 2.3 標準ロールの評価

現在の標準ロールは、初期導入や少人数運用では利用しやすい構成です。ただし、承認者と作成者を厳密に分ける会社では、標準ロールをそのまま使わず、職務別ロールを追加することを推奨します。

| ロール | 初期設定としての評価 | 厳格運用時の注意 |
| --- | --- | --- |
| システム管理者 | 妥当です。管理者向けの全操作権限です。 | 付与対象を最小限にしてください。通常ユーザーには付与しません。 |
| 受注管理責任者 | 妥当です。自部門の受注参照・作成・更新を行えます。 | 請求・入金は含まれないため、経理処理を担当する場合は別ロールを付与します。 |
| 発注管理責任者 | 小規模運用では妥当です。購買依頼から発注登録、支払予定参照まで確認できます。 | 作成・承認・更新が同一ロールに含まれるため、内部統制を重視する場合は依頼者、承認者、購買担当に分けます。検収は標準では含めず、必要に応じて検収担当ロールを追加します。 |
| 経理担当者 | 妥当です。全社の請求・入金・支払予定を処理できます。 | 全社参照を含むため、経理部門以外には付与しないでください。 |
| 一般メンバー | 初期利用には妥当です。自部門の受注作成と購買依頼作成ができます。 | 受注と購買依頼を別担当にしたい場合は、受注作成担当ロールと購買依頼者ロールに分けます。 |

結論として、標準ロールは「最初に動かすための推奨構成」としては合理的です。マニュアルを役割別に作成する場合は、標準ロールを基本形として説明し、厳格な承認分離が必要な会社向けに追加ロール例を併記してください。

### 2.4 業務口径で見たロール利用方針

標準ロールをそのまま利用する場合と、業務分掌に合わせて追加ロールを作る場合では、ユーザーに説明する内容が変わります。運用確認時は、以下の口径で整理してください。

| 業務上の役割 | 標準ロールで対応する場合 | 厳格に分ける場合の推奨ロール | 利用者に見せるべき範囲 |
| --- | --- | --- | --- |
| 営業担当者 | 一般メンバー | 受注作成担当 | 自部門の受注一覧、新規受注作成 |
| 営業責任者 | 受注管理責任者 | 受注更新担当 | 自部門の受注一覧、受注詳細、下書き編集、受注確定 |
| 請求担当者 | 経理担当者 | 請求担当 | 全社の受注参照、請求一覧、請求作成、請求取消 |
| 入金担当者 | 経理担当者 | 入金担当 | 全社の売掛台帳、回収管理、回収登録、回収取消 |
| 購買依頼者 | 一般メンバー | 購買依頼者 | 自部門の購買依頼一覧、新規購買依頼作成、申請 |
| 購買依頼承認者 | 発注管理責任者 | 購買依頼承認者 | 自部門の購買依頼一覧、承認、差戻し |
| 購買担当者 | 発注管理責任者 | 発注案件担当 | 発注案件一覧、発注案件作成、発注登録、仕入先参照 |
| 発注承認者 | 発注管理責任者 | 発注案件承認者 | 発注案件詳細、承認、差戻し |
| 検収担当者 | 標準ロールでは対応しない | 検収担当 | 発注一覧、納品・検収一覧、検収登録 |
| 支払担当者 | 経理担当者 | 支払担当 | 支払予定一覧、支払方法変更、支払済み更新 |
| システム管理者 | システム管理者 | システム管理者 | マスタ情報管理、ユーザー、部門、ロール権限 |

業務上の注意:

- 「一般メンバー」は、受注作成と購買依頼作成の両方を含む初期導入向けロールです。営業担当者と購買依頼者を分けたい会社では、そのまま利用しないでください。
- 「発注管理責任者」は、購買依頼から発注登録までを一人または少人数で管理するためのロールです。検収は標準では含めず、必要な会社では「検収担当」ロールを追加してください。
- 「経理担当者」は、全社参照と金銭処理を含むロールです。請求担当、入金担当、支払担当を分ける場合は専用ロールを作成してください。
- ロール別の画面表示やボタン表示は、[ロール別操作ガイド](./role-based-operation-guide.md) を参照してください。

## 3. 権限別の見え方

### 3.1 受注管理

| 権限 | 表示される主な画面 | できること |
| --- | --- | --- |
| 受注ダッシュボード表示 | ダッシュボード | 受注状況の確認 |
| 受注を全社参照 | 受注一覧、受注詳細 | 全社の受注を参照 |
| 自部門の受注を参照 | 受注一覧、受注詳細 | 所属部門の受注を参照 |
| 受注を作成 | 新規受注作成 | 受注を正式登録、自分の受注を確認 |
| 受注を更新 | 受注詳細、下書き編集 | 下書き保存、受注確定、受注編集 |
| 請求管理を表示 | 請求一覧、請求詳細 | 請求データを参照 |
| 請求を処理 | 請求作成、請求取消 | 請求作成、下書き請求削除、請求取消 |
| 売掛台帳を表示 | 売掛台帳 | 売掛残高を確認 |
| 入金を処理 | 回収登録、回収取消 | 入金登録、入金取消 |

注意:

- 受注作成のみのユーザーは、受注を正式登録できますが、下書き保存や登録後の再編集はできません。
- 請求・入金担当者に全社参照を付与しない場合、処理対象は本人または担当部門の受注に限定されます。
- 受注承認フローは現在未使用です。受注承認用の予約権限は通常付与しません。

### 3.2 発注管理

| 権限 | 表示される主な画面 | できること |
| --- | --- | --- |
| 発注ダッシュボード表示 | ダッシュボード | 発注状況の確認 |
| 発注を全社参照 | 発注案件・発注一覧・詳細 | 全社の発注案件、発注、納品・検収、仕入、支払予定を参照 |
| 自部門の発注を参照 | 発注案件・発注一覧・詳細 | 所属部門の発注案件、発注、納品・検収、仕入、支払予定を参照 |
| 購買依頼を作成 | 新規購買依頼 | 購買依頼を作成、自分の依頼を確認 |
| 購買依頼を更新・申請 | 購買依頼一覧 | 下書き保存、差戻し後の再申請 |
| 購買依頼を承認・差戻し | 購買依頼一覧 | 申請中の購買依頼を承認・差戻し |
| 発注案件を作成 | 新規発注案件 | 発注案件を作成、自分の案件を確認 |
| 発注案件を更新 | 発注案件詳細 | 発注案件更新、発注登録、仕入計上 |
| 発注案件を承認・差戻し | 発注案件詳細 | 申請中の発注案件を承認・差戻し |
| 仕入先管理を表示 | 仕入先一覧・詳細 | 仕入先情報を参照 |
| 検収を処理 | 納品・検収管理 | 納品予定作成、検収登録 |
| 支払予定を表示 | 支払予定一覧・詳細 | 支払予定を参照 |
| 支払予定を処理 | 支払予定一覧 | 支払方法変更、支払済み更新 |

注意:

- 購買依頼作成のみのユーザーは、申請作成と自分の依頼確認ができます。下書き保存や再編集には更新権限が必要です。
- 発注参照のみのユーザーには、購買依頼一覧は表示されません。購買依頼を確認する場合は、購買依頼作成、更新・申請、承認・差戻しのいずれかを付与します。
- 発注案件作成のみのユーザーは、案件作成と自分の案件確認ができます。下書き保存や再編集には更新権限が必要です。
- 検収のみのユーザーは、納品・検収までは処理できますが、検収後の仕入計上はできません。
- 支払予定のみのユーザーは、支払予定を処理できます。発注案件や検収の詳細確認まで必要な場合は、別途参照ロールを追加してください。

### 3.3 マスタ情報管理

| ロール | 表示される主な画面 | できること |
| --- | --- | --- |
| システム管理者 | テナント・ユーザー、部門、ロール権限 | 現在のテナント内のユーザー、部門、ロール、権限を管理 |
| スーパー管理者 | すべてのテナント | 全テナントを管理 |

注意:

- 通常のシステム管理者は、現在ログインしているテナントだけを管理できます。
- 全テナントを管理できるのは、システム側でスーパー管理者として指定されたアカウントだけです。
- 通常のシステム管理者は、保護されたスーパー管理者アカウントを変更できません。
- 最後のシステム管理者ロールを削除することはできません。

## 4. 権限組み合わせの注意

マスタ情報管理では、ロール権限の組み合わせに注意が必要な場合、画面上に警告が表示されます。

![権限組み合わせの注意](./assets/master/master-permission-warning.png)

主な警告例:

| 表示内容 | 意味 | 推奨対応 |
| --- | --- | --- |
| 作成のみ許可されています | 作成はできるが、下書き保存や再編集ができない | 必要に応じて更新権限を追加 |
| 検収後の仕入計上はできません | 検収担当と仕入計上担当が分かれている | 同じ担当者に任せる場合は発注案件更新を追加 |
| 互換権限と個別権限が重複しています | 旧形式と新形式の権限が混在 | 新規ロールでは個別権限を優先 |

## 5. マニュアル記載時の推奨表現

一般ユーザー向けマニュアルでは、内部権限コードではなく、業務名で説明してください。

例:

- 良い表現: 「請求を作成する権限があるユーザー」
- 避ける表現: 「内部権限コードを持つユーザー」

管理者向け資料では、必要に応じて権限コードを併記して構いません。
